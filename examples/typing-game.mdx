## Overview

The Typing Game is a multiplayer typing speed game where players compete in real-time to type passages of text as quickly and accurately as possible. It demonstrates a full-stack Foldkit application with:

- **Client** — Foldkit application with routing, RPC commands, and subscriptions
- **Server** — Effect RPC server managing rooms, games, and player state
- **Shared** — Type-safe RPC definitions and schemas shared between client and server

## Architecture

### Model Structure

The application uses a nested model with route-driven pages:

```typescript
const Model = S.Struct({
  route: AppRoute,
  home: Home.Model.Model,
  room: Room.Model.Model,
})
```

Each route has its own isolated model, message type, update function, and view.

### Game Flow

1. **Home Page** — Create or join a room
2. **Room Page** — Wait for players, start game, play, view results
3. **Game Phases** — Waiting → GetReady → Countdown → Playing → Finished

### RPC Integration

The client communicates with the server using Effect RPC:

```typescript
export const createRoomRpc = Rpc.make('createRoom', {
  payload: S.Struct({ username: S.String }),
  success: RoomAndPlayer,
})

export const subscribeToRoomRpc = Rpc.make('subscribeToRoom', {
  payload: S.Struct({ roomId: S.String, playerId: S.String }),
  success: RoomWithPlayerProgress,
  error: RoomNotFoundError,
  stream: true,
})
```

### Real-Time Updates

The room page subscribes to server updates using a streaming RPC subscription that pushes room state changes to all connected clients in real time.

## Key Features

### Multiplayer Rooms

Players create or join rooms identified by unique IDs. The room host can start the game when ready.

### Game Phases

The game progresses through distinct phases:

- **Waiting** — Players joining the room
- **GetReady** — Brief preparation phase
- **Countdown** — 3-2-1 countdown before game starts
- **Playing** — Players type the passage as fast as possible
- **Finished** — Display scoreboard with WPM and accuracy

### Progress Tracking

As players type, their progress is sent to the server and broadcast to other players in real-time, enabling live progress visualization.

### Scoring

When the game finishes, the server calculates:

- **WPM** (Words Per Minute) — Typing speed
- **Accuracy** — Percentage of correctly typed characters
- **Characters Typed** — Total characters entered

## Running the Example

The typing game requires both client and server to run:

```bash
# Terminal 1: Start the server
pnpm dev:typing-game:server

# Terminal 2: Start the client
pnpm dev:typing-game:client
```

Then open `http://localhost:5173` in multiple browser windows to test multiplayer functionality.

## What Makes This Example Special

### Full-Stack Type Safety

The shared RPC schemas ensure complete type safety between client and server. The client can't call a non-existent RPC, and the server can't return data that doesn't match the schema.

### Model-Driven Subscriptions

The subscription to room updates is managed by Foldkit's subscription system, which automatically starts and stops the stream based on the current route and model state.

### Nested Architecture

Each page (Home, Room) has its own isolated Model, Message, Update, and View. The root update function delegates to page-specific update handlers based on the current route.

### Real-Time State Sync

Changes to room state (new players joining, game starting, progress updates) are instantly synchronized across all connected clients through the streaming RPC subscription.
