Foldkit provides a collection of headless UI components that handle state management, keyboard navigation, ARIA attributes, and animations. All components follow the Model-View-Update pattern and integrate with the Foldkit runtime.

## Component Overview

- [Dialog](#dialog) — Modal dialog with native `<dialog>` element
- [Menu](#menu) — Dropdown menu with typeahead search
- [Tabs](#tabs) — Tab group with automatic or manual activation
- [Listbox](#listbox) — Single or multi-select listbox
- [Combobox](#combobox) — Autocomplete input with dropdown
- [Popover](#popover) — Floating panel with trigger button
- [Switch](#switch) — Toggle switch with accessible attributes
- [Disclosure](#disclosure) — Expandable content section

## Shared Patterns

All UI components share common patterns:

### Initialization

Every component exports an `init` function that creates the initial model:

```typescript
import * as Dialog from 'foldkit/ui'

const dialogModel = Dialog.init({ id: 'my-dialog' })
```

### Update Function

The `update` function processes messages and returns `[model, commands]`:

```typescript
const [nextModel, commands] = Dialog.update(model, message)
```

### View Function

The `view` function renders the component as `Html`:

```typescript
const element = Dialog.view({
  model,
  toMessage: identity,
  // ... component-specific config
})
```

### Lazy View

The `lazy` function creates a memoized view:

```typescript
const renderDialog = Dialog.lazy({
  panelContent,
  panelClassName: 'dialog-panel',
  backdropClassName: 'dialog-backdrop'
})

const element = renderDialog(model, identity)
```

## Dialog

Modal dialog component backed by the native `<dialog>` element with `showModal()`.

### Model

```typescript
type Model = {
  id: string
  isOpen: boolean
}
```

### Messages

<ParamField path="Opened" type="Message">
  Opens the dialog. Triggers the `showModal()` command.
</ParamField>

<ParamField path="Closed" type="Message">
  Closes the dialog via Escape key, backdrop click, or programmatically. Triggers the `closeModal()` command.
</ParamField>

<ParamField path="NoOp" type="Message">
  Placeholder for command completion.
</ParamField>

### init

```typescript
const model = Dialog.init({
  id: 'dialog-1',
  isOpen: false
})
```

<ParamField path="id" type="string" required>
  Unique identifier for the dialog.
</ParamField>

<ParamField path="isOpen" type="boolean" default="false">
  Initial open state.
</ParamField>

### view

```typescript
Dialog.view({
  model,
  toMessage: identity,
  panelContent,
  panelClassName: 'dialog-panel',
  backdropClassName: 'dialog-backdrop',
  className: 'dialog-wrapper'
})
```

<ParamField path="model" type="Model" required>
  Dialog model.
</ParamField>

<ParamField path="toMessage" type="(message: Closed | NoOp) => Message" required>
  Message mapper.
</ParamField>

<ParamField path="panelContent" type="Html" required>
  Content for the dialog panel.
</ParamField>

<ParamField path="panelClassName" type="string" required>
  CSS class for the panel.
</ParamField>

<ParamField path="backdropClassName" type="string" required>
  CSS class for the backdrop.
</ParamField>

<ParamField path="className" type="string">

### Helpers

## Menu

Dropdown menu with typeahead search, keyboard navigation, and drag-to-select support.

### Model

```typescript
type Model = {
  id: string
  isOpen: boolean
  isAnimated: boolean
  isModal: boolean
  transitionState: TransitionState
  maybeActiveItemIndex: Option<number>
  activationTrigger: 'Pointer' | 'Keyboard'
  searchQuery: string
  searchVersion: number
  maybeLastPointerPosition: Option<{ screenX: number; screenY: number }>
  maybeLastButtonPointerType: Option<string>
  maybePointerOrigin: Option<{ screenX: number; screenY: number; timeStamp: number }>
}
```

### Messages

<ParamField path="Opened" type="{ maybeActiveItemIndex: Option<number> }">
  Opens menu. `None` for pointer, `Some` for keyboard.
</ParamField>

<ParamField path="Closed" type="Message">
  Closes menu via Escape or backdrop click.
</ParamField>

<ParamField path="ClosedByTab" type="Message">
  Closes menu via Tab key.
</ParamField>

<ParamField path="ActivatedItem" type="{ index: number; activationTrigger: ActivationTrigger }">
  Highlights an item via arrow keys or hover.
</ParamField>

<ParamField path="DeactivatedItem" type="Message">
  Mouse leaves an enabled item.
</ParamField>

<ParamField path="SelectedItem" type="{ index: number }">
  Item selected via Enter, Space, or click.
</ParamField>

<ParamField path="MovedPointerOverItem" type="{ index: number; screenX: number; screenY: number }">
  Pointer moves over item.
</ParamField>

<ParamField path="Searched" type="{ key: string; maybeTargetIndex: Option<number> }">
  Printable character typed for typeahead.
</ParamField>

<ParamField path="ClearedSearch" type="{ version: number }">
  Search debounce period elapsed.
</ParamField>

### init

```typescript
const model = Menu.init({
  id: 'menu-1',
  isAnimated: true,
  isModal: false
})
```

<ParamField path="id" type="string" required>
  Unique identifier.
</ParamField>

<ParamField path="isAnimated" type="boolean" default="false">
  Enables CSS transition coordination.
</ParamField>

<ParamField path="isModal" type="boolean" default="false">
  Locks page scroll and inerts other elements when open.
</ParamField>

### view

```typescript
Menu.view({
  model,
  toMessage: identity,
  items: ['Option 1', 'Option 2', 'Option 3'],
  itemToConfig: (item, { isActive, isDisabled }) => ({
    className: isActive ? 'menu-item-active' : 'menu-item',
    content: div([], [item])
  }),
  buttonContent: span([], ['Open Menu']),
  buttonClassName: 'menu-button',
  itemsClassName: 'menu-items',
  backdropClassName: 'menu-backdrop'
})
```

<ParamField path="model" type="Model" required>
  Menu model.
</ParamField>

<ParamField path="toMessage" type="(message) => Message" required>
  Message mapper.
</ParamField>

<ParamField path="items" type="ReadonlyArray<Item>" required>
  Array of items to render.
</ParamField>

<ParamField path="itemToConfig" type="(item, context) => ItemConfig" required>
  Function mapping item to its configuration. Context includes `isActive` and `isDisabled`.
</ParamField>

<ParamField path="buttonContent" type="Html" required>
  Button content.
</ParamField>

<ParamField path="buttonClassName" type="string" required>
  Button CSS class.
</ParamField>

<ParamField path="itemsClassName" type="string" required>
  Items container CSS class.
</ParamField>

<ParamField path="backdropClassName" type="string" required>
  Backdrop CSS class.

<ParamField path="isItemDisabled" type="(item, index) => boolean">
  Predicate to mark items as disabled.
</ParamField>

<ParamField path="itemToSearchText" type="(item, index) => string">
  Extract search text from item. Defaults to item value.
</ParamField>

<ParamField path="isButtonDisabled" type="boolean">
  Disables the menu button.
</ParamField>

<ParamField path="itemsScrollClassName" type="string">
  CSS class for scrollable items wrapper.
</ParamField>

<ParamField path="className" type="string">

<ParamField path="itemGroupKey" type="(item, index) => string">

```typescript
  toMessage: identity,
  toView: ({ button, label, description, hiddenInput }) =>

      div(button, [
        span([Class('switch-thumb')], [])

      div(label, [span([], ['Enable feature'])]),
      div(description, [span([], ['Description text'])]),
      input(hiddenInput, [])

```

## TransitionState

Shared transition state type used by animated components:

```typescript
type TransitionState =
  | 'Idle'
  | 'EnterStart'
  | 'EnterAnimating'
  | 'LeaveStart'
  | 'LeaveAnimating'
```

Components with `isAnimated: true` coordinate CSS transitions using data attributes:

- `data-closed` — Applied during `EnterStart` and `LeaveAnimating`
- `data-enter` — Applied during `EnterStart` and `EnterAnimating`
- `data-leave` — Applied during `LeaveStart` and `LeaveAnimating`
- `data-transition` — Applied during all transition states

Use these attributes in your CSS to define enter/leave animations:

```css
.menu-items {
  opacity: 1;
  transition: opacity 200ms;
}

.menu-items[data-closed] {
  opacity: 0;
}
```
