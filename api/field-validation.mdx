The field validation module provides utilities for managing form field state and validation rules in Foldkit applications.

## Overview

Field validation in Foldkit uses a tagged union to represent field states (`NotValidated`, `Validating`, `Valid`, `Invalid`) and provides composable validation functions for common validation rules.

## makeField

Creates a tagged union of field states for a given value schema.

```typescript
import { Schema as S } from 'effect'
import { makeField } from 'foldkit/fieldValidation'

const EmailField = makeField(S.String)
```

### Returns

An object containing:

- `NotValidated` - Schema for field that hasn't been validated yet
- `Validating` - Schema for field currently being validated
- `Valid` - Schema for field with valid value
- `Invalid` - Schema for field with validation errors
- `Union` - Union schema of all field states
- `validate(validations)` - Function that returns first validation error
- `validateAll(validations)` - Function that returns all validation errors

### Example

```typescript
const EmailField = makeField(S.String)

// Create field states
const notValidated = EmailField.NotValidated({ value: '' })
const valid = EmailField.Valid({ value: 'user@example.com' })
const invalid = EmailField.Invalid({ 
  value: 'bad', 
  errors: ['Invalid email address'] 
})

// Use in model
const Model = S.Struct({
  email: EmailField.Union,
})

// Validate field
const validate = EmailField.validate([required(), email()])
const result = validate('user@example.com') // Valid({ value: ... })
```

## Types

### ValidationMessage

An error message for a validation rule — either a static string or a function that receives the invalid value.

```typescript
type ValidationMessage<T> = string | ((value: T) => string)
```

### Validation

A tuple of a predicate function and error message used for field validation.

```typescript
type Validation<T> = [Predicate.Predicate<T>, ValidationMessage<T>]
```

## Complete Example

```typescript
import { Schema as S } from 'effect'
import { 
  makeField, 
  required, 
  email, 
  minLength,
  type Validation 
} from 'foldkit/fieldValidation'

// Create field schemas
const EmailField = makeField(S.String)
const PasswordField = makeField(S.String)

// Define validation rules
const emailValidations: ReadonlyArray<Validation<string>> = [
  required('Email is required'),
  email('Please enter a valid email'),
]

const passwordValidations: ReadonlyArray<Validation<string>> = [
  required('Password is required'),
  minLength(8, 'Password must be at least 8 characters'),
]

// Create model
const Model = S.Struct({
  email: EmailField.Union,
  password: PasswordField.Union,
})

// Validate fields
const validateEmail = EmailField.validate(emailValidations)
const validatePassword = PasswordField.validateAll(passwordValidations)

// Use in update function
const emailResult = validateEmail('user@example.com')
// Valid({ value: 'user@example.com' })

const passwordResult = validatePassword('short')
// Invalid({ value: 'short', errors: ['Password must be at least 8 characters'] })
```
