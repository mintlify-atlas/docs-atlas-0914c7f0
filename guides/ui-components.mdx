## Overview

Foldkit provides a collection of headless UI components that handle complex interaction patterns, accessibility, and keyboard navigation. These components integrate seamlessly with Foldkit's Elm Architecture, using model-update-view patterns.

All components are **headless** — they provide behavior and accessibility but no styling, giving you complete design freedom.

## Available Components

Foldkit includes components for common UI patterns:

- **Dialog** — Modal dialogs with backdrop and focus management
- **Disclosure** — Expandable/collapsible content sections
- **Listbox** — Searchable, keyboard-navigable selection lists
- **Combobox** — Autocomplete input with dropdown
- **Menu** — Dropdown menus with keyboard navigation
- **Popover** — Floating content panels
- **Switch** — Toggle switches
- **Tabs** — Tab panels with keyboard support

## Basic Usage Pattern

All components follow the same pattern:

1. **Import** the component from `foldkit/ui`
2. **Initialize** the component's model with `init()`
3. **Handle messages** in your update function with `update()`
4. **Render** the component with `view()`

## Dialog Component

Dialogs create modal overlays with focus trapping and accessible ARIA attributes.

### Setup

```typescript
import { Schema as S } from 'effect'
import { Dialog } from 'foldkit/ui'
import { html } from 'foldkit/html'

const Model = S.Struct({
  confirmDialog: Dialog.Model
})

type Model = typeof Model.Type

const init = (): Model => ({
  confirmDialog: Dialog.init({ id: 'confirm-dialog', isOpen: false })
})
```

### Messages

```typescript
const OpenedConfirmDialog = m('OpenedConfirmDialog', Dialog.Opened)
const ClosedConfirmDialog = m('ClosedConfirmDialog', Dialog.Closed)
const DialogNoOp = m('DialogNoOp', Dialog.NoOp)

type DialogMessage = 
  | typeof OpenedConfirmDialog.Type
  | typeof ClosedConfirmDialog.Type
  | typeof DialogNoOp.Type
```

### Update

```typescript
const update = (model: Model, message: Message) =>
  M.value(message).pipe(
    M.tagsExhaustive({
      // When user clicks "Delete" button
      ClickedDelete: () => [
        evo(model, {
          confirmDialog: prevDialog => {
            const [nextDialog] = Dialog.update(prevDialog, Dialog.Opened())
            return nextDialog
          }
        }),
        []
      ],
      
      // Map dialog messages to app messages
      OpenedConfirmDialog: msg => {
        const [nextDialog, commands] = Dialog.update(
          model.confirmDialog,
          msg
        )
        return [
          evo(model, { confirmDialog: () => nextDialog }),
          commands.map(mapDialogMessage)
        ]
      },
      
      ClosedConfirmDialog: msg => {
        const [nextDialog, commands] = Dialog.update(
          model.confirmDialog,
          msg
        )
        return [
          evo(model, { confirmDialog: () => nextDialog }),
          commands.map(mapDialogMessage)
        ]
      }
    })
  )

const mapDialogMessage = (dialogMsg: Dialog.Message): Message =>
  M.value(dialogMsg).pipe(
    M.tagsExhaustive({
      Opened: () => OpenedConfirmDialog(dialogMsg),
      Closed: () => ClosedConfirmDialog(dialogMsg),
      NoOp: () => DialogNoOp(dialogMsg)
    })
  )
```

### View

```typescript
const { button, div, h2, p, Class, OnClick } = html<Message>()

const view = (model: Model): Html => {
  const dialogContent = div(
    [],
    [
      h2(
        [Class('text-xl font-bold mb-4')],
        ['Confirm Delete']
      ),
      p(
        [Class('mb-6')],
        ['Are you sure you want to delete this item?']
      ),
      div(
        [Class('flex gap-3')],
        [
          button(
            [
              Class('px-4 py-2 bg-red-500 text-white rounded'),
              OnClick(mapDialogMessage(Dialog.Closed()))
            ],
            ['Delete']
          ),
          button(
            [
              Class('px-4 py-2 bg-gray-200 rounded'),
              OnClick(mapDialogMessage(Dialog.Closed()))
            ],
            ['Cancel']
          )
        ]
      )
    ]
  )
  
  return Dialog.view({
    model: model.confirmDialog,
    toMessage: mapDialogMessage,
    panelContent: dialogContent,
    panelClassName: 'bg-white rounded-lg p-6 max-w-md',
    backdropClassName: 'fixed inset-0 bg-black bg-opacity-50'
  })
}
```

### ARIA Attributes

Dialogs automatically handle:
- `aria-labelledby` — Use `Dialog.titleId(model)` for your title element
- `aria-describedby` — Use `Dialog.descriptionId(model)` for your description

```typescript
const dialogContent = div(
  [],
  [
    h2(
      [Id(Dialog.titleId(model.confirmDialog)), Class('text-xl font-bold')],
      ['Confirm Delete']
    ),
    p(
      [Id(Dialog.descriptionId(model.confirmDialog)), Class('mb-6')],
      ['Are you sure you want to delete this item?']
    )
  ]
)
```

## Disclosure Component

Disclosures create expandable/collapsible sections with keyboard support.

### Setup

```typescript
import { Disclosure } from 'foldkit/ui'

const Model = S.Struct({
  faqDisclosure: Disclosure.Model
})

const init = (): Model => ({
  faqDisclosure: Disclosure.init({ id: 'faq-1', isOpen: false })
})
```

### View

```typescript
const view = (model: Model): Html => {
  return Disclosure.view({
    model: model.faqDisclosure,
    toMessage: mapDisclosureMessage,
    buttonClassName: 'w-full text-left px-4 py-3 bg-gray-100 hover:bg-gray-200',
    buttonContent: 'What is Foldkit?',
    panelClassName: 'px-4 py-3 bg-white',
    panelContent: 'Foldkit is a type-safe framework for building web applications...'
  })
}
```

### Lazy Views

For better performance with many components, use lazy views:

```typescript
const lazyDisclosureView = Disclosure.lazy({
  buttonClassName: 'w-full text-left px-4 py-3',
  buttonContent: 'FAQ Question',
  panelClassName: 'px-4 py-3',
  panelContent: 'Answer content...'
})

// In your view
lazyDisclosureView(model.faqDisclosure, mapDisclosureMessage)
```

The lazy view only re-renders when `model` or `toMessage` change.

## Listbox Component

Listboxes provide keyboard-navigable selection lists with search.

### Setup

```typescript
import { Listbox } from 'foldkit/ui'

type Theme = 'light' | 'dark' | 'system'

const themes: ReadonlyArray<Listbox.ItemConfig<Theme>> = [
  { id: 'light', value: 'light', label: 'Light' },
  { id: 'dark', value: 'dark', label: 'Dark' },
  { id: 'system', value: 'system', label: 'System' }
]

const Model = S.Struct({
  themeListbox: Listbox.Model,
  selectedTheme: S.String
})

const init = (): Model => ({
  themeListbox: Listbox.init({
    id: 'theme-selector',
    items: themes,
    orientation: Listbox.Orientation.Vertical
  }),
  selectedTheme: 'system'
})
```

### View

```typescript
const view = (model: Model): Html => {
  return Listbox.view({
    model: model.themeListbox,
    toMessage: mapListboxMessage,
    buttonClassName: 'px-4 py-2 bg-white border rounded',
    buttonContent: ({ item }) => item.label,
    listClassName: 'absolute mt-1 bg-white border rounded shadow-lg',
    itemClassName: ({ isActive }) => 
      `px-4 py-2 cursor-pointer ${isActive ? 'bg-blue-100' : ''}`,
    itemContent: ({ item }) => item.label,
    onSelect: ({ item }) => SelectedTheme({ theme: item.value })
  })
}
```

### Keyboard Navigation

Listboxes automatically support:
- **Arrow keys** — Navigate items
- **Home/End** — Jump to first/last item
- **Type-ahead** — Search items by typing
- **Enter/Space** — Select item
- **Escape** — Close listbox

## Multi-Select Listbox

For selecting multiple items:

```typescript
import { Listbox } from 'foldkit/ui'

const init = (): Model => ({
  tagsListbox: Listbox.Multi.init({
    id: 'tags-selector',
    items: tagItems,
    selectedIds: ['javascript', 'typescript'],
    orientation: Listbox.Orientation.Vertical
  })
})

Listbox.Multi.view({
  model: model.tagsListbox,
  toMessage: mapListboxMessage,
  // ... other config
  itemClassName: ({ isSelected, isActive }) =>
    `px-4 py-2 ${
      isSelected ? 'bg-blue-500 text-white' : ''
    } ${
      isActive ? 'ring-2 ring-blue-300' : ''
    }`
})
```

## Performance Tips

### Use Lazy Views

For components that don't change often:

```typescript
const lazyDialogView = Dialog.lazy({
  panelContent: dialogContent,
  panelClassName: 'bg-white rounded-lg p-6',
  backdropClassName: 'fixed inset-0 bg-black bg-opacity-50'
})

// Only re-renders when model or toMessage change
lazyDialogView(model.dialog, mapDialogMessage)
```

### Message Mapping

Create reusable message mappers:

```typescript
const mapDialogMessage = (msg: Dialog.Message): Message =>
  M.value(msg).pipe(
    M.tagsExhaustive({
      Opened: () => OpenedDialog(msg),
      Closed: () => ClosedDialog(msg),
      NoOp: () => DialogNoOp(msg)
    })
  )
```

## Accessibility

All components follow WAI-ARIA best practices:

- **Keyboard navigation** — Full keyboard support
- **Screen readers** — Proper ARIA labels and roles
- **Focus management** — Focus trapping in modals
- **State announcements** — Expanded/collapsed states

### Custom ARIA Labels

Add custom labels for better screen reader support:

```typescript
Disclosure.view({
  model: model.disclosure,
  toMessage: mapMessage,
  buttonClassName: 'w-full text-left',
  buttonContent: html => 
    div(
      [AriaLabel('Expand frequently asked questions')],
      ['FAQ']
    ),
  // ...
})
```

## Component API Reference

### Dialog

- `init(config)` — Create dialog model
- `update(model, message)` — Process dialog messages
- `view(config)` — Render dialog
- `lazy(staticConfig)` — Create memoized view
- `titleId(model)` — Get title element ID
- `descriptionId(model)` — Get description element ID

### Disclosure

- `init(config)` — Create disclosure model
- `update(model, message)` — Process disclosure messages
- `view(config)` — Render disclosure
- `lazy(staticConfig)` — Create memoized view

### Listbox

- `init(config)` — Create listbox model
- `update(model, message)` — Process listbox messages
- `view(config)` — Render listbox
- `lazy(staticConfig)` — Create memoized view
- `Orientation.Vertical` / `Orientation.Horizontal` — Layout direction
- `Multi.init()` / `Multi.view()` — Multi-select variant
